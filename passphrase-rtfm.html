Passphrase plugin


Syntax

function(parameter1,parameter2,parameter3);

1st parameter

A string; ElementID of input field to be monitored.

2nd parameter

A string; ElementID of field to display password tips.

3rd parameter

A 2 digit INT; greater than 09 & less than 50, these digits dictate which meter level will be used and which character combinations are 'required' to validate.
The first digit (1-4) signifies the meter to use, the second digit (0-9) denotes the required complexity of the passphrase.

    Possible Conditions
  • a. More than 5 characters
  • b. Mix of both uppercase and lowercase letters
  • c. Numerical character
  • d. Special character; $%^& etc.
A 3rd parameter of value '27', will use meter level 2 and have requirements a, b & d, (more than 5 characters including upper and lower case letters and a special character).
A 3rd parameter of value '40', will use meter level 4 but have no password requirements.

Example

<input type="password" id="passphrase" autocomplete="off" placeholder="Enter password..." value="" oninput="checkstr('passphrase','reqfield',42)">

In this example 'passphrase' is the ID of the textarea containing the password and 'reqfield' is the ID of a DIV showing the error message below the password field. The final field with value '42' sets the Meter level (AKA the color scheme) to #4 and the password requirements to level #2.
